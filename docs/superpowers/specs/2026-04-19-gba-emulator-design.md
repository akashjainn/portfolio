# GBA In-Portfolio Emulator — Design Spec
**Date:** 2026-04-19

## Overview

Embed a playable GBA emulator directly in the `adventuretime-gba` journal entry using EmulatorJS (self-hosted). The emulator renders inside the existing styled GBA shell component, auto-loads on mount, and shows touch controls on mobile or a button-tile keyboard guide on desktop.

---

## Decisions

| Topic | Decision |
|---|---|
| Shell style | Styled GBA housing (existing design in `GBAViewport.tsx`) |
| Load behavior | Auto-load on mount — no click-to-play gate |
| Emulator engine | EmulatorJS self-hosted (`@emulatorjs/emulatorjs`) |
| Mobile controls | EmulatorJS built-in virtual gamepad (`EJS_VirtualGamepad = true`) |
| Desktop controls | Button tile UI below the shell — visual GBA buttons with key label underneath |

---

## Architecture

### 1. Static asset setup

- Install `@emulatorjs/emulatorjs` as a dev dependency
- Add a `postinstall` script to `package.json` that copies `node_modules/@emulatorjs/emulatorjs/data/` → `public/emulatorjs/`
- The ROM already exists at `public/assets/adventuretime.gba` — no changes needed
- `public/emulatorjs/` is added to `.gitignore` (generated files, reproduced by `postinstall`)

### 2. `GBAViewport.tsx` (replaces current placeholder)

`'use client'` component. Accepts `artifact: JournalArtifact` (existing interface, unchanged).

**Rendering:**
- Renders the GBA shell div immediately with a `id="gba-player"` screen area
- Detects touch device via `window.matchMedia('(pointer: coarse)')` on mount
- On desktop: renders button tile keyboard guide below the shell
- On mobile: nothing extra — EmulatorJS virtual gamepad handles it

**EmulatorJS initialization (in `useEffect`):**
```
window.EJS_player = '#gba-player'
window.EJS_core = 'gba'
window.EJS_gameUrl = '/assets/adventuretime.gba'
window.EJS_pathtodata = '/emulatorjs/'
window.EJS_VirtualGamepad = true
window.EJS_Buttons = { playPause: false, restart: true, mute: true, settings: false, fullscreen: false }
```
Then append a `<script src="/emulatorjs/loader.js">` tag to `document.body`.

**Cleanup (return from `useEffect`):**
- Remove the appended script tag
- Delete the `EJS_*` globals from `window`

### 3. `GBAEmulator.tsx`

Delete — unused stub referencing `emulatorjs.com` external iframe.

### 4. `PressedSpecimen.tsx`

No changes. Already maps `artifact.type === 'viewport'` → `GBAViewport`.

### 5. `next.config.mjs`

- Remove `https://www.emulatorjs.com` from `frame-src` (no longer used)
- All other CSP entries stay — `'unsafe-eval'` already present for WASM, `script-src 'self'` covers `/emulatorjs/loader.js`

## Button Tile Keyboard Guide (desktop only)

Rendered below the GBA shell. Visual tiles matching GBA button colors:

| Tile | GBA button | Key |
|---|---|---|
| D-pad icon | D-pad | Arrow keys |
| Red tile "A" | A button | Z |
| Blue tile "B" | B button | X |
| Grey tile "START" | Start | Enter |
| Grey tile "SELECT" | Select | Shift |
| Grey tile "L" / "R" | L / R shoulders | A / S |

Tiles use the same red (`#8B2E1A`) and blue (`#2B4A8A`) already in the shell component.
